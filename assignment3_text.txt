Problem 1- VAE Synthetic Data with Low-Data Stabilization 
Using only 350 real examples per digit, train a conditional VAE. 
Before training the VAE: 
1. Augment the 350 real examples per digit by 10–20 times using rotations, shifts, small scaling, and light 
noise. 
2. Train the conditional VAE using the real + augmented data. 
3. For each digit, sample latent vectors: z∼N(0,I) 
4. Generate 5× (1000 examples in each run) of synthetic samples per digit. Each run should start with 
different random values of the weights, that should give better variants of generated samples. 
5. Pass generated samples through the LeNet-5 (trained on only the same allowed 350 examples) classifier 
and calculate: 
 confidence=max (softmax output of the trained LeNet above) 
6. Create three synthetic datasets: 
o 
Set A: all generated samples 
o 
Set B: high-confidence samples: confidence ≥ 0.9 
o 
Set C: mid-confidence samples: 0.6 ≤ confidence ≤ 0.9 
7. Train LeNet-5 using: 
o 
350 real examples per digit 
o 
plus, the selected VAE-generated samples 
8. Compare Set A, Set B, and Set C with: 
o 
the 350-real baseline 
o 
the 1000-real baseline 
o 
the GAN-based results 
Question: 
Does VAE-generated data benefit more from high-confidence selection or mid-confidence/diverse latent samples, 
and how does it compare with GAN-generated data and simple augmentation?

Then, build a second version of the CNN that includes a spatial attention mechanism. Compare the two models 
in terms of accuracy and training time. What difference does the attention mechanism make? 
b. Revisit the spoken digits task from Assignment 2, where you used spectrogram images to recognize 10 spoken 
digits. Build two CNN models for this task: one with an attention mechanism and one without (which you 
implemented in ass-2). Compare their performance and explain how attention affects the accuracy and training 
time in this case.

Problem 4: General Information Retrieval (including Q/A) 
Information Retrieval and Question Answering using Arabic language. This problem introduces classical 
keyword-based retrieval, semantic search using embeddings, and Retrieval-Augmented Generation 
(RAG). You will apply these techniques using a single Arabic book (in plain text format) as the reference 
source. 
Tasks 
(You must use a single open-source small LLM from HuggingFace — e.g., a model in the range of 1B–
3B parameters — throughout all tasks.) 
1. Book Preparation for Search 
• 
Select a public domain or permitted Arabic book in plain text format.  
• 
Be sure that the selected book was not used last year. 
• 
Split the text into short paragraphs (2–4 sentences).  
• 
Generate embeddings using a multilingual or Arabic sentence embedding model.  
• 
Index the vectors using a tool such as FAISS.  
2. Retrieval System 
• 
Build an interface that accepts Arabic queries.  
• 
Implement:  
o (a) Classical search (TF-IDF or BM25)  
o (b) Semantic search (embedding-based)  
• 
Display the top 5 results from both methods side by side for each query.  
3. RAG System 
• 
Extend the interface to answer Arabic questions using the same selected small LLM.  
• 
For each query:  
o 
(a) Retrieve relevant text using semantic search  
o 
(b) Generate an answer using RAG (LLM + retrieved content)  
o 
(c) Generate an answer using the LLM only (without retrieval)  
• 
Display both answers for comparison.  
Deliverables 
1. Description of the selected book:  
o Title and source  
o Text preprocessing and chunking method  
o Embedding and indexing process  
2. Results from at least 10 Arabic queries using:  
o Classical search  
o Semantic search  
3. Answers for 10 queries using:  
o RAG  
o LLM-only  
4. Comparison and reflection:  
o Which retrieval method produced more relevant results  
o Whether retrieval improved LLM-generated answers  
o Conclusions on the value of semantic retrieval and RAG for Arabic text  
 
Important Note 
• 
The model should be downloadable from HuggingFace and runnable locally (CPU or limited 
resources).  
• 
Choosing a less common book topic is encouraged, as it better highlights the benefit of retrieval.  
• 
Create different kind of questions including direct and indirect, easy and hard ones to get good 
insights. 
 
Problem 5: Benchmarking Arabic NLP Tasks Across LLMs 
Learning Objective 
The goal of this assignment is to evaluate how different Large Language Models (LLMs) perform on Arabic NLP 
tasks, to identify their strengths and weaknesses, and to understand the specific challenges of processing Arabic, 
including dialect variation, morphology, and ambiguity. 
Models to be Evaluated: ChatGPT, Gemini, ALLaM, Jais, Fanar 
Task Assignment: Each group is assigned one unique task from the list of 27 Arabic NLP tasks.  
1. Scope Definition: Each group must clearly define the task objective (such as accuracy, robustness, or 
hallucination), specify the application domain, and describe the data sources used. 
The dataset must include a balanced mix of Arabic varieties: 
• 
approximately one third of Modern Standard Arabic 
• 
one third Classical Arabic 
• 
one third Arabic dialects 
2. Dataset Creation: Each group must construct a dataset according to the predefined number of cases for the 
assigned task. 
Data requirements: 
• 
Real data should form the majority of the dataset 
Each sample must include a correct reference (gold) output. 
Annotation: 
• 
At least two annotators per sample 
• 
Disagreements resolved by majority voting (a 3rd annotator when needed) 
• 
Report on a simple inter-annotator agreement (IAA) measure 
The dataset must be clean, balanced, and well documented. 
3. Evaluation Protocol: All five models must be evaluated using the same inputs and the same prompt format. 
Model settings should be kept as consistent as possible. 
Each group must record and submit inputs, prompts, outputs 
4. Evaluation Metrics: Each group must use the predefined metric for its task and report both quantitative results 
and brief qualitative observations. 
5. Error Analysis: Each group must analyze model failures by including: 
• 
at least three incorrect or weak outputs 
• 
at least three Arabic-specific challenges (e.g., dialect confusion, morphology, ambiguity) 
• 
at least two difficult or borderline cases 
Each case must be explained clearly. 
6. Results Presentation 
Each group must present: 
• 
a comparison table across the five models 
• 
at least one visualization (such as a bar chart) 
• 
Selected output examples 
7. Report Requirements 
Each group must submit a report of no less than eight pages including: 
• 
task definition 
• 
dataset creation 
• 
annotation process 
• 
evaluation setup 
• 
results 
• 
error analysis 
• 
limitations 
• 
final conclusions, comments and recommendations. 
Important Rules 
• 
Each group must work on a different task and must evaluate all five models. All datasets must be 
original, and full logs must be submitted. 
• 
Below is a guideline checklist to be sure that the Benchmarking process is reasonably done. Each 
team should include in the report what they could apply of this checklist.
